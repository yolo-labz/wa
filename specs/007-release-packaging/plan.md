# Implementation Plan: Release Packaging and Service Integration

**Branch**: `007-release-packaging` | **Date**: 2026-04-10 | **Spec**: [spec.md](spec.md)

## Summary

Add GoReleaser v2 config cross-compiling for 3 targets with rcodesign notarization, a GitHub Actions release workflow, a Nix flake, `wad install-service`/`uninstall-service` subcommands (launchd on macOS, systemd on Linux), `wa upgrade` (prints the right command), and CHANGELOG generation via git-cliff. No business logic changes. Config-heavy feature with ~400 LoC of Go.

## Technical Context

**Language/Version**: Go 1.25 (toolchain pinned in go.mod)
**New dependencies**: None in `go.mod`. GoReleaser, rcodesign, git-cliff are CI-only tools.
**Storage**: None new. Service files are generated on disk by `wad install-service`.
**Testing**: `goreleaser check` for config validation; `nix flake check`; `wad install-service --dry-run` + `plutil` on macOS; unit tests for upgrade detection.
**Constraints**: `CGO_ENABLED=0` everywhere (constitution IV). Never root for service install.
